如何處理 AI 任務異常

預計時程（每個任務都有三段完成時限）：

任務建立ClaudeCodex最終決策
盤前研究08:10≤ 08:25≤ 08:2908:30
開盤研究09:01≤ 09:25≤ 09:2909:30
盤中研究11:00≤ 11:15≤ 11:2511:30
盤後研究15:05≤ 15:25≤ 15:2915:30
明日策略18:10≤ 18:25≤ 18:2918:30

常見異常與處理：

  1. PENDING 過了預計 Claude 完成時間
    → Claude Code Agent 可能沒啟動或沒讀檔。打開 Claude Code 視窗，輸入：
    讀 D:\ai\stock\claude-research-request.json 並執行研究
  2. CLAUDE_DONE 卡住，Codex 沒接手
    → Codex runner 沒跑。檢查：
    ps aux | grep codex
    若沒跑，重啟 Codex watcher。
  3. CLAUDE_FAILED / CODEX_FAILED
    → 看「訊息」欄錯誤原因，常見為解析失敗或檔案損毀。
    → 按「⏭ 標為過期」→ 再手動建新任務或等下個週期。
  4. FINALIZED 沒進 → 通知沒發出
    → 到排程頁找對應 Notify Job（例 PremarketNotifyJob）按「▶ 手動」補發。
  5. 過了預計時間但全部 ✓
    → 只是延遲，不影響結果。

說明：系統有自動 sweep job（每 60 秒）會把 PENDING 超過 30 分鐘自動過期、CLAUDE_RUNNING 超過 20 分失敗、CODEX_RUNNING 超過 10 分失敗。
