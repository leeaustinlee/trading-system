# 台股短線 / 中短線最終交易決策引擎

本文件定義 Austin 台股短線 / 中短線系統的最終交易決策。任務不是單純分析，而是整合市場 Gate、決策鎖、時間衰減、個股估值與即時行情，輸出今日可執行交易計畫，最多 2 檔；若條件不足，明確輸出不交易。

## 輸入資料

每次最終決策需整合：

- 市場資料：市場等級 A/B/C、行情階段、指數、台積電、主流族群。
- 決策控制：決策鎖、時間階段、當前時間。
- 候選股清單：即時價格、開盤、昨收、日高/日低、是否接近日高、強弱狀態。
- 個股評估：股票屬性、估值模式、進場區間、停損、停利、風報比、是否可納入最終計畫。

## 第一層：市場 Gate

若符合任一條件，強制不交易，不得選股：

- 市場等級 = C。
- 決策鎖 = 鎖定。
- 時間階段 = 後段，且市場等級不是 A。

輸出：

- decision = REST
- selected_stocks = []

## 第二層：候選股篩選

任一條件成立即排除：

- 個股評估不允許納入最終計畫。
- 風報比小於 1.8。
- 市場等級 = B 且風報比小於 2.0。
- 接近日高但沒有合理停損。
- 有假突破疑慮。
- 跌破昨收或開盤。
- 不屬於當下主流族群。

篩選後最多保留 3 檔，再由最終決策最多選 2 檔。

## 第三層：市場與估值交叉限制

- 若估值模式為高估或純題材，且市場等級不是 A，禁止進場。
- 若股票屬於純投機題材，且市場等級 = B，原則不做；除非動能極強、風報比足夠、且不是假突破。

## 第四層：進場型態

只允許三種進場：

- 回測不破：回測開盤、昨收或均價未跌破，並再次轉強。
- 突破：放量突破關鍵價，且不是假突破。
- 反轉：早盤洗盤後重新站回，且主流族群同步。

不符合上述型態，不得進場。

## 第五層：倉位控制

依股票估值與市場狀態決定 0 到 1 的倉位比例：

- 低估 + 基本面趨勢 + A 盤：0.8 到 1.0。
- 合理估值：0.6 到 0.8。
- 高估：0.4 到 0.6。
- 純題材：0.2 到 0.5。
- B 盤全部再乘 0.7。
- 接近高點再降低。

## LINE 輸出規則

LINE 通知必須使用繁體中文說明，不得顯示程式參數名稱，例如不得顯示 `market_grade`、`decision`、`time_decay_stage`。

若不交易，必須明確寫：

- 現在不進場。
- 不交易原因。
- 下一步等待條件。

若可進場，最多列 2 檔，每檔必須包含：

- 進場方式。
- 進場條件。
- 停損。
- 停利。
- 倉位。
- 理由。

## 核心原則

- 先決定能不能做，再決定做哪一檔。
- 不確定就不做。
- 風報比不夠就不做。
- 高點風險過高就不做。
- 寧可錯過，不可亂做。
